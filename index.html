Air-Gapped Security Network

The BBA Platform lives on a dedicated security LAN physically isolated from all other networks — no shared infrastructure with corporate IT, guest Wi-Fi, or public internet.

Dedicated switches, cabling and patch panels — no shared hardware
No outbound internet route — firewall blocks all external traffic by default
10 GbE internal backbone — high throughput, low latency, fully private
VLAN segmentation per subsystem (VMS, ACS, Drone, BMS)
🪪

Zero-Trust Access Model

No user has more access than their role requires. Every session is authenticated, every action is attributed, and nothing can be deleted or modified without a trace.

Multi-factor authentication (MFA) — mandatory for all users
Role-based permissions — operator, supervisor, manager, read-only, audit
Automatic session timeout + re-authentication on sensitive actions
Immutable audit log — tamper-evident, SIRA-compliant, court-ready
Biometric login option for SCR workstations
🔐

Encryption Everywhere

All video, metadata, incident records and communications are encrypted — both at rest and in transit. No data ever leaves the building in readable form.

AES-256 encryption for all stored video and incident data
TLS 1.3 for all internal API communication between services
Encrypted database backups — keys stored separately from data
Evidence export packages are digitally signed and hash-verified
📡

Intrusion Detection & Response

The platform monitors itself. Unusual login patterns, unexpected process activity, and unauthorized device connections trigger internal alerts before any damage is done.

Network IDS — detects port scanning, ARP spoofing, rogue devices
Failed login lockout + security team alert after 3 attempts
Process integrity monitoring — unauthorized software execution blocked
Offline update policy — no live package manager, no auto-update
Periodic penetration testing protocol included in maintenance contract
🛡️

Hardware Failsafe — Software Cannot Disarm Physical Security

Even if the platform software is fully compromised, physical security remains intact. HVM barriers, emergency doors, and access control panels all have local manual override — they do not require the software to function. A cyberattack cannot open a barrier or unlock a restricted door. The attacker would gain visibility, not physical control.
